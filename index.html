Page Manager

Basically this is a graphic user interface to hook_menu that allows you to create new paths or take over existing system paths.

System paths include node/%nid, taxonomy/term/%tid, and user/%uid.
Using Page Manager helps you understand how Drupal builds pages via system paths and how that's different from URL.

Panels

http://drupal.org/project/panels

At its core it is a drag and drop content manager that lets you visually design a layout and place content within that layout.

What does Panels do?

  • Panels is a page rendering handler used within Page Manager.
  • Panels essentially replaces the theme rendering function of a given entity.
  • Panels replaces the $page['content'] variable in your template.
I'll start slightly out of my depth and bring it closer to home for many site builders and themers.
When Page Manager is enabled we can pass off the handler for a system path. DEMO variant type
When enabled with a typical Drupal theme, Panels works within the established theme regions. Panels takes over the $page['content'] variable

Layout Plugins

Provide your own custom layout within your theme.

theme.info

We need to tell our theme that it is providing layout plugins to Panels and where to find them within our theme directory.


; ========================================
; Panels & Page Manager
; ========================================

plugins[panels][layouts] = 'plugins/layouts'
Add a plugins directory to your theme.
Add these lines to your .info file

Style Plugins

Style plugins provide the opportunity to alter the output of Pane content.
Panels ships with a few of these: no markup, rounded corners, default.

theme.info

We need to tell our theme that it is providing style plugins to Panels and where to find them within our theme directory.


; ========================================
; Panels & Page Manager
; ========================================

plugins[panels][layouts] = 'plugins/layouts'
plugins[panels][styles] = 'plugins/styles'
Add a styles directory.
Add this line to your .info file.

Semantic Panels

http://drupal.org/project/semantic_panels

It's a CTools style plugin that gives you full control over the HTML output of Panels panes.

Semantic Panels is my favorite style plugin since it helps to remove lots of the nested divs and provide some projects specific HTML.
It is also exportable inside of Features.

Custom Content Panes

The module ctools_custom_content is a sub-module in Ctools.

Create custom, exportable, reusable content panes for applications like Panels.

This is essentially like a Block, but exportable and reusable in multiple places.
It uses input filters and provides an admin overview screen.
You can use token content from the Panels contexts for display.
DEMO: Chrome

Panels Everywhere

http://drupal.org/project/panels_everywhere

Panels Everywhere is an advanced method to completely do away with Drupal's restrictive blocks system and instead use the much more freeing Panels Layout system to control how your pages look. Panels Everywhere modifies the page as it is being rendered to 'wrap' the content in a display and can even take over your page theme to do away with the need for a page.tpl.php.

Panels Everywhere takes over the page rendering.
